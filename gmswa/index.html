The intuition

Retrieval and state tracking are different jobs

Most long-context benchmarks emphasize retrieval. GMSWA targets the other job: maintaining a changing hidden state.

⌕
Retrieval

Find something that already exists

A fact, a name, a needle, a copied token. Sliding-window attention can still be a strong local evidence engine.

↻
State tracking

Update what is true over time

A bit flips, an entity changes, a plan advances, a task phase moves forward. The model needs a state that persists.

GMSWA's thesis: local attention retrieves; recurrent memory tracks state; the gate decides which engine to trust.
The mechanism

The memory was not broken. The gate was.

The key discovery is simple and showable: naive hybrids learn the local shortcut and under-use memory. A memory-first gate initialization fixes the routing.

Sliding-window attention

Good at nearby evidence, local composition, and retrieval cues.

+

Recurrent memory

Carries a compact state across the full sequence; β∈(0,2) enables flip-like updates.

→

Memory-aware gate

Start by trusting memory, then learn when local evidence or state memory should dominate.
